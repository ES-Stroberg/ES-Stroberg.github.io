Character Movement

Design Intention: Movement

Short animation times, low cooldowns, and freedom to combo moves contributes to better control of the character. Movement becomes second nature, which ultimately contributes to a fast-paced experience.

    void OnDash()
    {
        if (dashCooldownLeft <= 0 && lifeCycle.isAlive && canMove && !MenuManager.isPaused)
        {
            //Activates timer for when the dash trail vfx should end
            StartCoroutine(dashTrailTime());

            //Disables "dash ready" HUD element
            if (dashIndicator)
                dashIndicator.SetActive(false);

            //Transitions into dash animation pose
            if (animator)
                animator.SetBool("isDashing", true);

            //Plays dash VFX
            if (vfxHandler)
            {
                vfxHandler.psDash.Play();
                vfxHandler.psDashTrail.Play();
            }

            //Plays dash SFX
            SoundManager.Playsound("dash");

            //Drag is modified to make the dash accelarate and decelerate along a pleasant curve
            rigidbody.drag = 1;

            //Keeps track on when player can dash again
            dashCooldownLeft = dashCooldownTime;

            //Keeps track on when the dash has ended
            dashTimeLeft = dashTime;

            //The actual dash
            rigidbody.AddForce(moveDirection * dashForce, ForceMode.Impulse);
        }
    }

Character Attacks

Hit Detection

Ray Casting: Top View

The hit detection was relatively primitive. It drew a number of rays in a cone around the player character.

Problem: Sloped Surfaces

The ray-tracing proved problematic once sloped surfaces and elevations were introduced.

Problem: Small Props

The ray-tracing proved problematic once again when small props were added.

Side View

The rays were originally drawn from the base of the character, which worked fine on flat surfaces.

Solution: More Rays

The problem was simply solved by drawing additional rays from the top of the characters.

Solution: Even More Rays

Why make it complicated?

    int angle = (int)(hitArcAngle / 2) * -1;
    rayNumber = (int)hitArcAngle / 5;
    for (int i = 0; i < rayNumber; i++)
    {
        Vector3 forwardDirection = Quaternion.Euler(0, angle, 0) * transform.forward;
        angle += 5;

        Vector3 topPos = new Vector3(transform.position.x, transform.position.y + 1.5f, transform.position.z);

        RaycastHit[] hits = Physics.RaycastAll(topPos, forwardDirection, maxAttackLength);
        for (int j = 0; j < hits.Length; j++)
        {
            if (!hits[j].collider.isTrigger)
            {
                IKnockback knockbackEffect = hits[j].collider.GetComponent<IKnockback>();
                if (knockbackEffect != null)
                {
                    Vector3 knockback = transform.forward * knockBackForce * attackCharge;
                    knockbackEffect.Knockback(gameObject, knockback);
                }
                break;
            }
        }
        Debug.DrawLine(topPos, topPos + forwardDirection * maxAttackLength, Color.red, 2);
    }
