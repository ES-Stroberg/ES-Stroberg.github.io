Game Overview

Player Goal

Don't get crushed, stick together and embrace each other through life and death. Skrot is a cooperative game where the goal is to stay alive for as long as possible to rack in the highest score.

Design Pillar

Physics Based Movement

Character Movement

The Anatomy

The character consists of eight extendable pistons pointing in different directions and a static (hehe) screen.

Controller Input

The piston that pushes is relative to the rotation to the gamepads' left stick. When the stick is pointing upwards it will push the currently upwards facing piston. If the (A) button is held, the piston will be pushed further.

The Logic: Revisited

The original logic of the movement is based on a lot of work-arounds and over-complicated solutions. So I decided to revisit the code and rethink the logic. Here's how I would do it today.

Each piston has a local up-vector,

and is stored into an array

A function loops trough the array of pistons and compares the input vector to the pistons' respective up-vector

The stick input is converted into a 3D vector

The function normalizes the vectors and returns a reference to the piston with the smallest angle difference
